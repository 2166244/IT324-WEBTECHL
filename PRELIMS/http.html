Note 1

Version History:

HTTP 0.9 (The One-line Protocol, 1991)

- It is the initial version of the HTTP which has a single-line request nature, supported by the GET method only, and respond with a hypertext, and the connection is immediately cut after the response.

- Does not support HTTP headers, URL's, Version, Error Codes. HTTP 1.0 (RFC 1945 Building extensibility, May 1996)

- Friendly with browsers protocol - Supports header fields which includes metadata with both request and response message. - The response is unlimited to hypertext. - It supports the GET, HEAD, POST methods - The connection is immediately cut after the response.

TERMINOLOGIES

1. Connection - Established connections among different programs for communication

2. Message – HTTP’s basic unit of communication which transmit syntax with the use of connection.

3. Request – refers to how HTTP request a message, HTTP request message

4. Response – refers to how HTTP respond a message, HTTP response message

5. Resource – network object identifying by the URL

6. Entity- represent different resources such as data, service

7. Client – application program for sending request

8. User agent – initiates a request with the use of editors, browser, etc.

9. Server – accepts request and sends back responses.

10. Origin Server – where resources are created

11. Proxy – client-side portals which handle requests with the use of protocols

12. Gateway – server-side portals and used as translate protocols to access resources

13. Tunnel – intermediate program between two connections

14. Cache – local storage for message, retrieval, and deletion of response messages
